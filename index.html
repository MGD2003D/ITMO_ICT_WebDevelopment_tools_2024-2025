    Таблица user

    Поле Тип PK FK Описание
    book_id integer Да book.id ID книги
    tag_id integer Да tag.id ID тега

    Поле Тип PK FK Описание
    id integer Да — Первичный ключ
    name string Нет — Уникальное имя тега, индексируется

    Связи:
    - books — Many-to-Many → Book.tags (через BookTagLink)

    Таблица book

    Поле Тип PK FK Описание
    id integer Да — Первичный ключ
    title string Нет — Название книги
    description string? Нет — Описание книги (опционально)
    owner_id integer Нет user.id Владелец книги

    Связи:
    - owner — Many-to-One → User.books
    - requests — One-to-Many → ExchangeRequest.book
    - tags — Many-to-Many → Tag.books (через BookTagLink)

    Таблица exchangerequest

    Поле Тип PK FK Описание
    id integer Да — Первичный ключ
    book_id integer Нет book.id Ссылка на книгу
    requester_id integer Нет user.id Кем сделан запрос
    owner_id integer Нет user.id Владелец книги
    status enum Нет — Статус запроса: pending, accepted, rejected
    created_at datetime Нет — Время создания запроса

    Связи:
    - book — Many-to-One → Book.requests
    - requester — Many-to-One → User.sent_requests
    - owner — Many-to-One → User.received_requests

class Tag(SQLModel, table=True):

    Реализация пользовательского функционала (15 баллов)

    В рамках задания на 15 баллов был полностью реализован базовый модуль аутентификации и управления пользователями. Ниже приведено описание ключевых компонентов и конечных точек.

    1. Схемы данных (schemas.py)

    • UserCreate – модель входных данных для регистрации (username, password).
    • UserRead – выходная модель с публичными полями пользователя (id, username).
    • Token и TokenData – модели для работы с JWT (access_token, token_type, необязательное поле username в полезной нагрузке).
    • PasswordChange – входные данные для смены пароля (old_password, new_password).
    Содержимое файла `schemas.py`

    2. Аутентификация и JWT (security.py)

    • Используется Passlib с алгоритмом bcrypt для безопасного хэширования паролей.
    • Функции:
    • hash_pwd(pwd: str) → str — хэширование пароля.
    • verify_pwd(pwd, hashed) → bool — проверка соответствия пароля и хэша.
    • create_access_token(data: dict, expires_min: int) — генерация JWT с полем exp.
    • decode_token(token: str) — декодирование и валидация JWT.
    • Все секреты (ключ шифрования JWT) загружаются через .env (переменная KEY).
    Содержимое файла `security.py` 
    from passlib.context import CryptContext
from datetime import datetime, timedelta
from jose import jwt, JWTError
import os
from dotenv import load_dotenv

load_dotenv()

SECRET_KEY = os.getenv("KEY")
ALGORITHM = "HS256"
ACCESS_TOKEN_MIN = 30

pwd_ctx = CryptContext(schemes=["bcrypt"], deprecated="auto")


def hash_pwd(pwd: str) -> str:
    return pwd_ctx.hash(pwd)


def verify_pwd(pwd: str, hashed: str) -> bool:
    return pwd_ctx.verify(pwd, hashed)


def create_access_token(data: dict, expires_min: int = ACCESS_TOKEN_MIN):
    to_encode = data.copy()
    exp = datetime.utcnow() + timedelta(minutes=expires_min)
    to_encode.update({"exp": exp})
    return jwt.encode(to_encode, SECRET_KEY, algorithm=ALGORITHM)


def decode_token(token: str):
    return jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])

    3. Маршруты аутентификации (auth.py)

    Маршруты сгруппированы под префиксом /auth и снабжены тегом auth.

    1. Регистрация
      POST /auth/register
    2. Принимает UserCreate (login + пароль).
    3. Проверяет уникальность пользователя.
    4. Хэширует пароль и сохраняет нового User в базу.

    5. Возвращает UserRead.

    6. Логин и получение токена
      POST /auth/login

    7. Использует стандартную схему OAuth2PasswordRequestForm (поля username и password).
    8. Проверяет учётные данные через verify_pwd.

    9. При успешной аутентификации генерирует и возвращает JWT в формате Token.

    10. Получение текущего пользователя
      GET /auth/me

    11. Извлекает токен из заголовка Authorization: Bearer <token>.
    12. Декодирует и проверяет JWT, получает sub (имя пользователя).

    13. Загружает пользователя из базы и возвращает UserRead.

    14. Список всех пользователей
      GET /auth/users

    15. Требует действующий JWT (Depends(get_current_user)).

    16. Возвращает массив UserRead для всех пользователей в системе.

    17. Смена пароля
      POST /auth/change-password

    18. Принимает PasswordChange с old_password и new_password.
    19. Проверяет старый пароль через verify_pwd.
    20. Если верно — обновляет поле hashed_password на новый хэш и сохраняет изменения.
    21. Возвращает статус 204 No Content.
    Содержимое файла `auth.py` 
    from fastapi import APIRouter, Depends, HTTPException, status, Form
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
from sqlmodel import Session, select
from schemas import UserCreate, UserRead, Token, TokenData, PasswordChange
from models import User
from connection import get_session
from security import hash_pwd, verify_pwd, create_access_token, decode_token
from jose import JWTError

router = APIRouter(prefix="/auth", tags=["auth"])
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/login")


def get_user_by_name(session: Session, username: str) -> User | None:
    return session.exec(select(User).where(User.username == username)).first()


def authenticate(session: Session, username: str, password: str) -> User | None:
    user = get_user_by_name(session, username)
    if user and verify_pwd(password, user.hashed_password):
        return user


@router.post("/register", response_model=UserRead, status_code=201)
def register(payload: UserCreate, session: Session = Depends(get_session)):
    if get_user_by_name(session, payload.username):
        raise HTTPException(400, "username taken")
    u = User(username=payload.username,
             hashed_password=hash_pwd(payload.password))
    session.add(u)
    session.commit(); session.refresh(u)
    return u


@router.post("/login", response_model=Token)
def login(form: OAuth2PasswordRequestForm = Depends(),
          session: Session = Depends(get_session)):
    user = authenticate(session, form.username, form.password)
    if not user:
        raise HTTPException(status_code=401, detail="invalid creds")
    token = create_access_token({"sub": user.username})
    return {"access_token": token, "token_type": "bearer"}


def get_current_user(token: str = Depends(oauth2_scheme),
                     session: Session = Depends(get_session)) -> User:
    try:
        payload = decode_token(token)
        username: str = payload.get("sub")
        assert username
    except (JWTError, AssertionError):
        raise HTTPException(401, "Could not validate credentials")
    user = get_user_by_name(session, username)
    if not user:
        raise HTTPException(401, "User not found")
    return user


@router.get("/me", response_model=UserRead)
def me(current: User = Depends(get_current_user)):
    return current


@router.get("/users", response_model=list[UserRead])
def list_users(session: Session = Depends(get_session),
               _: User = Depends(get_current_user)):
    return session.exec(select(User)).all()


@router.post("/change-password", status_code=204)
def change_password(data: PasswordChange,
                    current: User = Depends(get_current_user),
                    session: Session = Depends(get_session)):
    if not verify_pwd(data.old_password, current.hashed_password):
        raise HTTPException(400, "old password incorrect")
    current.hashed_password = hash_pwd(data.new_password)
    session.add(current); session.commit()

    4. Механизм работы

    1. Регистрация создаёт запись в таблице user с уже зашифрованным паролем.
    2. Логин выдаёт клиенту JWT, который содержит поле "sub": username и время жизни.
    3. Аутентификация на защищённых маршрутах (/auth/me, /auth/users, /auth/change-password) происходит через зависимость Depends(oauth2_scheme) и функцию get_current_user, которая:
    4. Декодирует токен,
    5. Проверяет срок годности и целостность подписи,
    6. Загружает пользователя из базы,
    7. Бросает ошибку 401, если что-то пошло не так.
    8. Смена пароля защищена той же зависимостью и не допускает доступа без корректного JWT.

    Реализован полный цикл: - Регистрация → Логин → Получение токена → Доступ к защищённым ресурсам → Смена пароля. Все операции оформлены в отдельных маршрутах, снабжены Pydantic-схемами и соответствуют REST-контракту приложения.

def run_migrations_offline() -> None:
    """Run migrations in 'offline' mode.

    This configures the context with just a URL
    and not an Engine, though an Engine is acceptable
    here as well.  By skipping the Engine creation
    we don't even need a DBAPI to be available.

    Calls to context.execute() here emit the given string to the
    script output.

    """
    url = config.get_main_option("sqlalchemy.url")
    context.configure(
        url=url,
        target_metadata=target_metadata,
        literal_binds=True,
        dialect_opts={"paramstyle": "named"},
    )

    with context.begin_transaction():
        context.run_migrations()


def run_migrations_online() -> None:
    """Run migrations in 'online' mode.

    In this scenario we need to create an Engine
    and associate a connection with the context.

    """
    load_dotenv()
    url_tokens = {
        "DB_USER": os.getenv("DB_USER", ""),
        "DB_PASS": os.getenv("DB_PASS", ""),
        "DB_HOST": os.getenv("DB_HOST", ""),
        "DB_NAME": os.getenv("DB_NAME", "")
    }

    url = config.get_main_option("sqlalchemy.url")

    url = re.sub(r"\${(.+?)}", lambda m: url_tokens[m.group(1)], url)
    connectable = create_engine(url)

    with connectable.connect() as connection:
        context.configure(
            connection=connection, target_metadata=target_metadata,
            compare_type=True,
            compare_server_default=True,
        )

        with context.begin_transaction():
            context.run_migrations()


if context.is_offline_mode():
    run_migrations_offline()
else:
    run_migrations_online()
